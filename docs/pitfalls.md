Project Pitfalls Checklist (Must Avoid)

Automatic penalties
- Using LangChain/LlamaIndex chains instead of hand-rolled retrieval: −15
- Eval set generated by an LLM instead of hand-written: −10
- No citations in generated answers: −10
- No public deployment URL at viva: −10

Common pitfalls to avoid
1. Skipping the eval set until Week 4 (must be ready by Week 2).
2. Trusting LLM-as-judge scores without human validation.
3. Chunking naively (fixed-size chunks split resume sections).
4. Ignoring retrieval failures (log retrieved chunks for every query).
5. Over-engineering before measuring (baseline recall@k first).
6. Treating fairness as an afterthought (model card section is graded).

Quality reminders
- Citations must appear in every generated answer.
- Use section-aware chunking (per experience entry / project).
- Validate judge vs human labels (Cohen’s kappa or Spearman).
- Keep a focused role family to avoid noisy evaluation.
